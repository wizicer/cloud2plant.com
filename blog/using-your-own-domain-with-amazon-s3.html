Using your own domain with Amazon S3

August 6, 2014

By now you probably know that you can host a static website on Amazon S3, but did you know you can use your own domain name too?

By default, you can access your S3-hosted website by going to:

  • bucket-name.s3.amazonaws.com or
  • s3.amazonaws.com/bucket-name

Here’s how to access it from www.your-domain.com.

Create your bucket

To use your own domain with Amazon S3, you’ll need to create a bucket that matches your domain/subdomain exactly. If your website is located at www.your-domain.com, you’ll need to create a bucket called www.your-domain.com. If you’re using a different subdomain, the same rule applies.

Keep in mind that bucket names have to be unique throughout the entire S3 platform. This shouldn’t be a problem since you’re using your own domain, but it’s a good thing to keep in mind.

Update your DNS

In your website’s DNS settings, create a CNAME record that points from www.your-domain.com to s3.amazonaws.com. It should look something like this:

Type: CNAME
Domain: www.example.com
Target: s3.amazonaws.com.

If your bucket is hosted in Europe, you may need to set the target to s3-external-3.amazonaws.com instead.

Once you’ve created the CNAME record, you’ll have to wait for the DNS to propagate. Be patient—it may take up to 24 hours.

A note about naked domains

If you’re using the www subdomain to host your website, you’ll probably want to setup forwarding from your naked domain example.com to www.your-domain.com. This will ensure visitors who don’t type in www will still end up at the right place.

Most hosting providers offer this service in their control panel. If you’re not sure how to configure it, ask your web host for assistance.

Go to your domain

Once the DNS has fully propagated, you should be able to visit www.your-domain.com in your browser and see your website. If applicable, make sure to check that your-domain.com is forwarding to the right place. If it’s not working, double check your CNAME record. Chances are, it’s not configured properly.

Of course, this trick isn’t limited to static websites running on S3. You can use it to access just about any public object stored in an S3 bucket. If your not hosting your whole site on S3, this is a great way to offload styles, scripts, images, videos, etc. and free up resources on your own web server.
